## 自定义指令扩展

为了满足大家定制专属微信执行口令（例如发送 `/pt_download` 返回下载状态）的需求，并且保证用户在未来仍能无损平滑更新官方 Docker 镜像，我们在接下来的版本中计划原生引入以下两套与主代码解耦的指令注入模式：

### 模式一：Webhook 双向触发接管（低代码架构 · 推荐 ⭐️）
这是最契合软路由、NAS 玩家以及智能家居生态的设计：
1. **触发拦截**：当输入不属于内置列表的指令时，Bridge 将不再静默，而是把该指令连同参数推送至您预配的 `WEBHOOK_URL`。
2. **外部运算**：您可以借用外部的工作流引擎（如 **Node-RED、FastGPT 或青龙面板**）去挂载识别这些特殊的 `/` 指令并执行关联动作。
3. **闭环回写**：外部业务动作跑通后，反向请求 WeChat-Bridge 的 `/api/send` 接口把操作结果直推回微信对话框。

### 模式二：YAML 静态动作字典挂载（零代码架构）
专门为非后端开发者量身打造：
1. **外置部署**：在外部 Docker 挂载点（如 `./data/`）维护一个命名约定的格式配置文件（如 `custom_commands.yml`）。
2. **编写简单规则**：
   - 静态回复：设定 `/wifi` 直接映射为回复字符串 `"SSID:xxx 密码:xxx"`。
   - 动态动作：设定 `/reboot` 执行触发一个隐式的 HTTP GET 回调地址。
3. **主程序扫描**：当匹配到对应口令时，主引擎在底层热读取阻断并调度分发，**真正实现免写一行 Python 代码的即插即用**。
